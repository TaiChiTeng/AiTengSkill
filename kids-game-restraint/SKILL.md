# 儿童益智游戏设计：克制原则

## Overview

面向低龄儿童（3-8岁）的益智游戏，核心设计原则是**克制**——减少干扰本身就是保护专注力。不做加法抢注意力，做减法留空间。

## When to Use

- 设计儿童益智游戏的动画、音效、UI 节奏
- 选择配色方案或视觉风格
- 设计交互方式（手势、按钮、拖拽等）
- 设定难度曲线或游戏时长
- 设计反馈机制（正确/错误/过关）

**When NOT to use:** 面向青少年或成人的游戏、非教育类纯娱乐游戏。

## 七维克制

### 1. 时间克制
- 单局 **2-5 分钟**，控制屏幕时间
- 游戏流程有明确开始和结束，不设计无限循环
- 不给家长增加管理屏幕时间的负担

### 2. 节奏克制
- 动画要**慢**，给足观察窗口
- 空间变化动画（旋转、翻转）≥ 1.5 秒
- 过渡用**缓出曲线**（ease-out），起步快结尾慢，让儿童看清最终位置
- 不用匀速或缓入
- 动画的“自然映射”原则：动画不能仅是装饰，更应该是**规则的可视化解释**。设计动画时，要先看游戏规则是怎样的，如何从视觉上让玩家接受这个规则，优先寻找与游戏规则高度契合的现实物理现象作为隐喻。比如**循环移位** → 可以考虑类似传送带 / 转盘 / 排队的动画设计：物体从一端离开视野，同时从另一端进入，因果链完整不中断

### 3. 刺激克制
- **不抢注意力**：不用倒计时、闪烁、弹窗、粒子特效
- 高亮用呼吸脉冲（正弦缓动），不用闪烁
- 提示文字用淡入淡出，不用弹跳或抖动
- 结算反馈逐步呈现（依次出现），不同时炸开
- 不制造紧迫感

### 4. 情绪克制
- 不催促、不制造焦虑、不惩罚
- 猜错不扣分、不重置进度，低成本试错
- 错误反馈温和（短暂低音），不是刺耳警报
- 鼓励探索，不强调"犯错"

### 5. 色彩克制
- **低饱和柔和色**，不用高饱和刺眼颜色
- 推荐风格：纪念碑谷式柔和暖色调
- 背景用渐变，不用纯色或高对比度
- 文字用高不透明度深色（如深棕 `rgb(88,65,48)`），可读但不刺眼

### 6. 交互克制
- 不用复杂手势：无双指缩放、无长按、无多点触控
- 点击/触摸是最主要交互方式
- 操作目标要大（按钮、卡片），容易点击
- 支持键盘但不依赖键盘

### 7. 文字量克制
- 界面文字**尽量少**
- 图标 + 文字双通道呈现，不识字也能玩
- 文字用于强化认知（图文关联），不是必需的信息通道
- 提示语简洁，不超过 10 个字

## Quick Reference

| 维度 | 要做 | 不要做 |
|------|------|--------|
| 时间 | 单局 2-5 分钟，有明确结束 | 无限循环，一局玩太久 |
| 节奏 | 动画慢（≥1.5s），缓出曲线 | 快速动画，匀速/缓入 |
| 刺激 | 呼吸脉冲，淡入淡出，逐步呈现 | 闪烁，弹窗，粒子，倒计时 |
| 情绪 | 低成本试错，温和反馈 | 惩罚，扣分，刺耳警报 |
| 色彩 | 低饱和柔和色，深色文字 | 高饱和，高对比度，荧光色 |
| 交互 | 点击/触摸，大按钮 | 复杂手势，小操作目标 |
| 文字 | 少文字，图标+文字，提示≤10字 | 大段文字，纯文字指引 |

## Difficulty Design

- 渐进难度：先建立信心，再逐步加码
- 每轮任务拆小，不一次要求太多
- 有限的抉择选项
- 选择难度动态调整：比如玩家选错，则使错误选项不再响应
- 不需要让最后一关也很难

## Sound Design

- Web Audio API 程序化生成，不依赖外部音频文件
- 正确反馈：上行短音阶（如 C-E-G），轻快愉悦
- 错误反馈：低频短音，温和不刺耳
- 旋转/过渡：短促白噪声或轻微机械声
- 过关：短琶音，适度庆祝
- 音量 0.05-0.12，不吓人

## Visual Style

- 推荐风格：纪念碑谷式低多边形（low-poly）
- `flatShading: true` 统一几何感
- 背景装饰物缓慢浮动，不静止也不快速运动
- 阴影柔和（PCF Soft Shadow）
- 毛玻璃风格 UI 浮层（`backdrop-filter: blur`）

## Red Flags — 出现以下情况时停下来审视

- 添加了倒计时或限时机制
- 动画时长 < 1 秒
- 使用了闪烁、抖动、弹跳效果
- 猜错后有惩罚性反馈
- 高饱和度颜色（`#FF0000`、`#00FF00` 等）
- 要求双指操作或长按
- 界面文字超过 2 行
- 音量 > 0.15
